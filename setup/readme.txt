###### 1. 本系统模块概述：

基本模块：综合介绍，新闻中心，部门介绍，产品图片，人物图片，其它图片，视频播放，
         下载列表，访客留言

内部模块：内部会员，内部公文，内部论坛

会员模块：会员中心，公共论坛，观点辩论，商务供求
          商务供求含：含 企业资料，企业介绍，新闻中心，供求交易，招聘信息；可扩展；


扩展模块：浮动广告，图片广告，文字广告，RSS订阅，投票调查，访问统计，DB管理器，
          档案查询，友情连接，站点导航，


工具模块：木马扫描，IP监控，超级素数，计数器，万年历，算24，算素数，连接抓取，
          鸽子年历，颜色选框，效率测试，阿江探针，进制转换，安全中心，ASCII码，
		  特殊字符，圆角效果，素数计算，

典型特征：多级类别，多语言版，会员发布，扩展模块，模板显示 ... 
		  内容存文件，FCK编辑器，使用缓存技术，utf-8编码，水印开关，图片认证码，
		  图片等比缩放，权限测试，js分页程序，js繁体转化

###### 2. 安装与配置

*** 2.1. 初始安装(setup)

A. 把文件复制到网站跟目录(/)或某一目录（如/shell/），以下用跟目录(/)作说明；
   如果下载得到的目录为以下两个目录： upfile/；root/； 则做如下处理：
   A: 小站模式(小型网站建议这样配置)
   A.1. 把root作为跟目录或把下面的所有文件（夹）复制到跟目录；
   A.2. 把upfile复制到跟目录；
   A.3. 把公共文件夹img复制到跟目录，公共文件夹asp_edfck复制到/sadm/改为edfck；

   B: 大站模式(大型网站建议这样配置)
   B.1. 把root作为跟目录或把下面的所有文件（夹）复制到跟目录；
   B.2. 把upfile虚拟目录；
   A.3. 把img设置成两个虚拟目录，执行权限可设置成无(不执行)；
   A.4. 把公共文件夹asp_edfck复制到/sadm/改为edfck，或设置成虚拟目录；
   B.5. 后台设置：刷新与杂项 >> 信息刷新 >> 数据转化/清理 >> 执行 
        [CFile: 数据转移DB-=>File ------ 设置数据表+年份]，和其它相关操作，
		此设置把内容存成文件格式，减轻数据库负担！
   B.6. 必要的话，也可把DB（数据库）设置成虚拟目录，或升级成MSSQL数据库； 
B. 修改/upfile/sys/config/Config.asp: 设置参数：Const Config_Path = "/" 根据你放置的目录设置；
C. 设置写权限：(整个网站)，安装好后可只给 /upfile/目录写权限；
D. 浏览器打开http://www.domain.com/setup/setup.asp，如果没有setup.asp文件，
   只有setup.aspPeace文件,则把此文件改为setup.asp即可；
   如果找不到目录，可能执行了G步骤，请到/sadm/setup/目录找找；
E. 根据提示安装配置,
F. 注意：
   F.1. 根据提示得到登陆后台地址，分别进入后台[系统与设置>>配置] 和 [系统与设置>>参数>>系统] 设置 
        更多详细参数,设置好后分别点 参数页的“刷新”和配置页的“刷新配置”生效。
   F.2. 在后台，进入[系统与设置>>工具>>安全处理与应用>>开发者说明文件]，得到详细的“开发者说明文件”。
   F.3. 请记录好你的设置值(如后台地址，管理帐号密码等)，并请删除目录setup；如果今后要重新安装，
        可以恢复本目录，并把setup.aspPeace文件改名为setup.asp后，重新运行即可。
G. 安全提示：
   安装后建议删除/setup/目录，或者把此目录移动到网站以外，至少/setup/下不能再有setup.asp文件
   系统与设置 >> 配置 >> 清理垃圾 >> 清理setup >> 可达到此目的

*** 2.2. 重新配置安装

A. 把文件复制到网站跟目录(/)或某一目录（/shell/），以下用跟目录(/shell/)作说明。
B. 修改/shell/upfile/sys/config/Config.asp: 设置参数：Const Config_Path = "/shell/" 
C. 设置写权限：/websa/upfile/目录
D. 设置好IIS，浏览器打开 /shell/sadm/login.asp 登陆
    (login.asp文件名可能根据后台安全设置而更改成别的文件名)
E. 后台执行：系统与设置 >> 配置 >> 系统路径 设置成/websa/并刷新。

*** 2.3. 初始密码 与 设置(适用于忘记密码)

A. 默认：帐号：webadmin，密码：sa1234（不含空格，区分大小写）；如果密码错误，请按以下方式操作
B. [忘记密码]推荐做法：打开文件/sadm/login.asp源代码 找到约13行，按提示设置；

###### 3. 网站初始化设置(栏目等内容)

3.1. 初始化设置: 第一次使用，先作如下设置工作吧：（以下项目如果找不到连接，请查看如下2.2. ）
A. 系统与设置 >> 数据库管理，先备份吧！如果需要，把文件都备份一下
B. 系统与设置 >> 模块，可先阅读如下3.2中得到的“开发者说明文件”3.1，3.2；
C. 系统与设置 >> 参数，根据需要设置； 
D. 系统与设置 >> 配置，自己慢慢阅读配置页下方资料；

3.2. 小技巧：
A. 刷新与杂项 >> 信息刷新 >> 管理超级连接 改成 
    ../../tools/help/vhelp.asp, 
    ../../tools/tools.asp?Act=Admin, 
    ../../sadm/admin/sys_config.asp, 分别提交，
    可以看到开发者说明文件，隐藏的默认菜单，配置文件等。
B. 如果有时为了简化用户操作，隐藏了一些菜单，但为了方便维护人员有时要用上而特别设置的超级连接。
C. 更多说明文件，请登陆后台：刷新与杂项 >> 信息刷新 >> 管理超级连接 改成 
    ../../tools/help/vhelp.asp 查看开发者说明文件
